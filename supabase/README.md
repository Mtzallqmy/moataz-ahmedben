# إعداد Supabase للإصدار الإنتاجي

1. افتح SQL Editor في مشروع Supabase ونفّذ الملف `supabase/schema.sql` مرة واحدة.
2. فعّل تأكيد البريد أو عطّله حسب إعدادات Auth التي تريدها.
3. أضف متغيرات `.env.example` في Vercel. استخدم `SUPABASE_SERVICE_ROLE_KEY` للخادم فقط، ولا تضعه في أي متغير يبدأ بـ `VITE_`.
4. أعد النشر بعد حفظ المتغيرات.

المشروع يستخدم Supabase Auth للمصادقة وRLS لعزل المستخدمين. مفاتيح مزودي الذكاء الاصطناعي تُرسل عبر HTTPS إلى وظائف Vercel وتُخزّن في `providers.encrypted_key` باستخدام AES-256-GCM. لا تُعاد المفاتيح إلى الواجهة ولا تُسجل في السجلات.

إذا كان لديك جدول `providers` قديم يحتوي `encrypted_data` فقط، أعد إضافة المزودات من صفحة المزودين بعد تشغيل المخطط؛ المفاتيح القديمة غير قابلة للاستخدام مع بنية التشفير الجديدة.
